3.响应response

回到上文提到的响应体：

参数含义：
finish_reason:有stop、length、function_call、content_filter、null

  1. stop: api返回了完整的信息；
  2. length: 说明max_tokens参数调的太小，或本身的token数不够用；
  3. function_call：模型决定调用函数
  4. content_filter: 由于内容过滤器的标记而省略了内容
  5. null: 仍在响应或者响应失败
